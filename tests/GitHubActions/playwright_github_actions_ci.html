Complete GitHub Actions workflow for Playwright API tests — secrets setup, matrix environments, HTML report upload, and PR annotations

What you need before writing the workflow

Three things must be in place before the workflow file makes sense.

1

npm test script in package.json

The workflow calls npm test. Make sure it maps to Playwright.

# package.json
{
  "scripts": {
    "test": "playwright test",
    "test:smoke": "playwright test --grep @smoke",
    "test:ci": "playwright test --reporter=html,list"
  }
}
2

playwright.config.ts is CI-aware

Retries, workers, and reporter differ between local and CI.

# playwright.config.ts
export default defineConfig({
  fullyParallel: true,
  retries: process.env.CI ? 2 : 0,
  workers: process.env.CI ? 4 : undefined,
  reporter: process.env.CI
    ? [['html', { open: 'never' }], ['github']]
    : 'list',
  use: {
    baseURL: process.env.BASE_URL,
    extraHTTPHeaders: {
      'Authorization': `Bearer ${process.env.API_TOKEN}`,
    },
  },
});
3

Workflow file location

Create .github/workflows/api-tests.yml at the repo root.

The github reporter annotates failing tests directly on PR lines in the GitHub UI — no need to open the HTML report to see which test failed. Add it alongside html in CI so you get both the annotation and the downloadable report.

Basic workflow — runs on every PR

The minimal working workflow. Gets smoke tests running on every pull request in under 2 minutes.

.github/workflows/api-tests.yml
name: API tests

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  test:
    name: Run API tests
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Set up Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Run smoke tests
        run: npx playwright test --grep @smoke
        env:
          BASE_URL: ${{ secrets.STAGING_BASE_URL }}
          API_TOKEN: ${{ secrets.STAGING_API_TOKEN }}

npm ci not npm install. In CI always use npm ci — it installs exact versions from package-lock.json, never resolves new versions, and errors if the lockfile is out of sync. This makes builds reproducible and prevents "works on my machine" surprises.

Secrets — the right way

Never hardcode tokens. All credentials live in GitHub Secrets and arrive in the workflow as environment variables.

Where to add secrets in GitHub
1

Repository secrets

Go to your repo → Settings → Secrets and variables → Actions → New repository secret

2

Environment secrets

For per-environment credentials: Settings → Environments → New environment → Add secret. Use staging and production environments.

3

Reference in workflow

Use ${{ secrets.SECRET_NAME }} — GitHub masks this value in all logs automatically.

Secrets to create for API testing
# Repository secrets — available to all workflows
STAGING_BASE_URL = https://staging-api.yourapp.com
STAGING_API_TOKEN = your-staging-token
STAGING_TEST_EMAIL = test@yourapp.com
STAGING_TEST_PASSWORD = testpassword123

# Production environment secrets — locked to prod env
PROD_BASE_URL = https://api.yourapp.com
PROD_API_TOKEN = your-prod-token
Using secrets in the workflow
steps:
  - name: Run tests
    run: npx playwright test
    env:
      BASE_URL: ${{ secrets.STAGING_BASE_URL }}
      API_TOKEN: ${{ secrets.STAGING_API_TOKEN }}
      TEST_EMAIL: ${{ secrets.STAGING_TEST_EMAIL }}
      TEST_PASSWORD: ${{ secrets.STAGING_TEST_PASSWORD }}
      CI: true

Never print secrets to logs. GitHub masks ${{ secrets.X }} values in workflow logs, but if you echo the value to a file or pass it through a custom command that logs its arguments, it may appear in plaintext. Treat secrets as write-only in workflow files.

HTML report — upload and access after a run

The HTML report is the most useful debugging artifact. Upload it so anyone can download and open it — even after tests fail.

Upload report as a workflow artifact
      - name: Run tests
        run: npx playwright test
        env:
          BASE_URL: ${{ secrets.STAGING_BASE_URL }}
          API_TOKEN: ${{ secrets.STAGING_API_TOKEN }}

      # if: always() — upload even when tests fail
      # Without this, a failed test cancels all subsequent steps
      - name: Upload HTML report
        uses: actions/upload-artifact@v4
        if: always()
        with:
          name: playwright-report-${{ github.run_id }}
          path: playwright-report/
          retention-days: 30

if: always() is mandatory on the upload step. Without it, GitHub skips all steps after the first failure. Your tests fail, the upload step is skipped, and there's no report to debug with. if: always() makes the upload unconditional — it runs whether tests passed, failed, or were cancelled.

How to access the report after a run
1

Open the workflow run on GitHub

Go to Actions → your workflow run

2

Scroll to Artifacts section

Download playwright-report-{run_id}.zip

3

Unzip and open locally

Run npx playwright show-report path/to/report

Full production workflow

Complete workflow with two jobs — smoke tests on every PR, full regression nightly — plus HTML report upload, caching, and environment matrix.

.github/workflows/api-tests.yml — complete
name: API tests

on:
  pull_request:
    branches: [main, develop]
  push:
    branches: [main]
  schedule:
    - cron: '0 2 * * *' # nightly at 02:00 UTC
  workflow_dispatch: # manual trigger from UI

jobs:

   # ── Job 1: smoke on every PR ─────────────────────
  smoke:
    name: Smoke tests (staging)
    runs-on: ubuntu-latest
    if: github.event_name == 'pull_request'

    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - run: npm ci

      - name: Smoke tests
        run: npx playwright test --grep @smoke
        env:
          CI: true
          BASE_URL: ${{ secrets.STAGING_BASE_URL }}
          API_TOKEN: ${{ secrets.STAGING_API_TOKEN }}

      - name: Upload report
        uses: actions/upload-artifact@v4
        if: always()
        with:
          name: smoke-report-${{ github.run_id }}
          path: playwright-report/
          retention-days: 7

   # ── Job 2: full regression nightly ───────────────
  regression:
    name: Full regression (${{ matrix.env }})
    runs-on: ubuntu-latest
    if: github.event_name == 'schedule' || github.event_name == 'workflow_dispatch'

    strategy:
      fail-fast: false # run all envs even if one fails
      matrix:
        env: [staging, production]

    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - run: npm ci

      - name: Regression — ${{ matrix.env }}
         # Production only runs smoke — never full write tests on prod
        run: |
          if [ "${{ matrix.env }}" = "production" ]; then
            npx playwright test --grep @smoke
          else
            npx playwright test --grep @regression
          fi
        env:
          CI: true
          BASE_URL: ${{ matrix.env == 'production' && secrets.PROD_BASE_URL || secrets.STAGING_BASE_URL }}
          API_TOKEN: ${{ matrix.env == 'production' && secrets.PROD_API_TOKEN || secrets.STAGING_API_TOKEN }}

      - name: Upload report — ${{ matrix.env }}
        uses: actions/upload-artifact@v4
        if: always()
        with:
          name: regression-${{ matrix.env }}-${{ github.run_id }}
          path: playwright-report/
          retention-days: 30

fail-fast: false on the matrix is critical. Without it, if staging tests fail, GitHub immediately cancels the production job too — so you never find out whether production is healthy. Always set fail-fast: false on environment matrices so every environment runs to completion independently.
